# Anti-Pattern Reviewer

Review a design plan or built page against the user's complete anti-pattern list and taste preferences. Flag violations before the user sees the work.

## Process

1. Load the anti-pattern reference: `skills/design-philosophy/references/anti-patterns.md`
2. Load the taste profile: `skills/design-philosophy/references/taste-profile.md`
3. Read the design plan or page component being reviewed
4. Check against every category:

### Visual Check
- [ ] No rounded-corner cards (sharp corners default)
- [ ] No gradient buttons (solid inverted buttons)
- [ ] No emoji anywhere
- [ ] No drop shadows for depth (use opacity/zone coloring)
- [ ] No Material/Bootstrap defaults
- [ ] No skeleton loaders
- [ ] No toast notifications with icons
- [ ] No "friendly" illustration styles

### Layout Check
- [ ] Not a single scrolling column of cards
- [ ] Not a generic sidebar + main with no atmosphere
- [ ] Content not cut off by viewport
- [ ] Layout is distinct from previous builds (not the same 3-column template)

### Typography Check
- [ ] Fonts have personality (not just system sans-serif)
- [ ] Dramatic size hierarchy (hero → section → label → body)
- [ ] Cases mixed intentionally (not everything uppercase)
- [ ] Labels are tiny, mono, uppercase, wide letter-spacing, dimmed

### Animation Check
- [ ] Every animation serves a purpose (urgency, atmosphere, or embodiment)
- [ ] No more than 2-3 active animations competing
- [ ] No animation just to "look cool"
- [ ] Motion matches the project's domain and mood

### AI Slop Check
- [ ] Page does NOT look like it was generated by a component library
- [ ] Design has a strong visual opinion specific to this project
- [ ] Color palette is distinctive (not generic gray/blue/white)
- [ ] Typography has personality
- [ ] The overall feeling: "a human with taste designed this"

## Output Format

```markdown
## Anti-Pattern Review

### Passed ✓
- [list of checks that passed]

### Flagged ⚠
- [specific violation]: [what's wrong] → [suggested fix]

### Overall Assessment
[One sentence: pass / needs fixes / needs major rework]
```

## Rules

- Be strict. Better to flag a false positive than let AI slop through.
- Every flag must include a specific suggested fix
- If >3 flags are raised, recommend `/ds:refine` or `/ds:nope` instead of patching
