# /ds:blowup — Experimental Redesign

Go wild. Break conventions. Try something that might fail spectacularly or might be brilliant.

## Process

1. Read the current page to understand the project
2. Do NOT ask for permission or preferences — just build something unexpected
3. Pick techniques, palettes, and layouts that haven't been tried before
4. Combine things that shouldn't go together
5. Build it, verify it renders, present it

## Inspiration

- Hot pink backgrounds (like Feral)
- Horizontal-only layouts
- Typography as the entire visual (like Kinetic)
- A single massive canvas visualization as the whole page
- Mix light and dark zones on the same page
- Use a technique from `animation-techniques.md` that hasn't been applied to this domain before
- Combine two incompatible themes (what if Cipherdeck met Stillwater?)

## Rules

- Only one rule: it must still be built around the project's content. Wild ≠ random.
- Do not ask the user before building. The point is surprise.
- If it doesn't work, that's fine. `/ds:nope` exists for a reason.
